# AGENTS.md

Orientation for coding agents working in this repo. Read this first so you don't
re-derive the architecture every session. For the manuscript-reconciliation
tooling, this file summarizes; **`HARNESS.md` is the authority**.

---

## What this is

A standalone **Jekyll 4** website for the book *The Greatest Shortcoming:
Quantitative Chauvinism and Lebensraum Imaginaries* (Brian C. Keegan). It is part
cinematic landing experience, part "network reading" of the book's chapters and
concepts. It deploys to **GitHub Pages** via GitHub Actions on push to `master`.

It is **not** a github-pages-gem site and uses **no client-side JS framework** —
the interactive pieces are hand-written vanilla JS.

## TL;DR for a new session

- Edit content/markup, then **`npm run build:css`** (Node-based Tailwind — works
  locally). **Do not** expect `bundle exec jekyll build` to work on this machine
  (Ruby gem/toolchain is broken locally); **CI is the real build/validation.**
- `assets/css/dist/main.css` is **gitignored and generated** — never hand-edit it;
  edit `assets/css/src/main.css` and rebuild. CI rebuilds it regardless.
- Site has two front-door pages: **`/` = landing** (cinematic intro), **`/home.html`
  = home hub** (title card + concepts + chapters). See *Routing* below.
- Nav lives **once** in `_config.yml` → `nav`, rendered by two includes.
- Several `_data/*.yml` files are **generated by the reconciliation harness** — see
  their header comments before editing by hand.
- Another Cowork session may mutate the tree mid-task — re-check `git status`
  before committing. Don't commit unless asked.

## Tech stack

| Layer | Choice | Notes |
|-------|--------|-------|
| Static site | **Jekyll ~4.4**, standalone (`Gemfile`) | NOT the `github-pages` gem — deliberately, so it builds on Ruby 4 |
| Ruby | **4.0** (CI) | Gemfile declares `csv`/`base64`/`webrick` explicitly for Ruby 4 |
| Markdown | **kramdown**, `input: GFM` | matches old github-pages rendering |
| Permalinks | `pretty` | pages override with explicit `permalink:` in front matter |
| Plugins | `jekyll-seo-tag`, `jekyll-sitemap`, `jekyll-redirect-from` | |
| CSS | **Tailwind CSS 3.4** via the Tailwind **Node CLI** + `@tailwindcss/typography` | source → dist; custom unlayered CSS at the bottom of the source file |
| JS | **vanilla** (no framework, no bundler) | `assets/js/landing.js` + small inline scripts |
| Hosting | **GitHub Pages** via Actions | `.github/workflows/github-pages.yml` |

## Build & local dev

```bash
npm ci                 # install Tailwind toolchain (Node 22 in CI)
npm run build:css      # assets/css/src/main.css -> assets/css/dist/main.css (--minify)
npm run dev:css        # same, --watch

# Jekyll build — works in CI (Ruby 4 + bundler-cache); typically BROKEN locally.
bundle exec jekyll build --destination ./_site --trace
```

**CI pipeline** (`.github/workflows/github-pages.yml`): checkout → Node 22 →
`npm ci` → `npm run build:css` → Ruby 4 (`bundler-cache`) → `jekyll build`
(`JEKYLL_ENV=production`) → upload artifact → **deploy** (only on non-PR events;
PRs build-only to catch breakage).

**Local validation when Jekyll won't run** (do these before pushing):
- `npm run build:css` — must compile without error.
- `node --check assets/js/landing.js` — JS syntax.
- `python3 bin/lint-site.py` — offline gate: parses `_config.yml` + every
  `_data/*.yml`, checks chapter/landing front matter, that every `cards.yml` slug
  resolves to a chapter file, and that no deprecated id/slug/alias survives.
- `python3 -m json.tool metadata/*.json` — JSON validity.
- Eyeball Liquid tag balance (`{% for %}/{% endfor %}`, `{% if %}/{% endif %}`).

## Routing / site map

| URL | Source | Layout | Purpose |
|-----|--------|--------|---------|
| `/` | `index.html` | `landing` | **Landing** — cinematic intro |
| `/home.html` | `home.html` | `home` | **Home hub** — title card + concepts + chapters |
| `/chapters/<slug>/` | `_chapters/NN-slug.md` | `chapter` | the 10 chapters (preface + 9) |
| `/about/` | `pages/about.md` | `default` | About (renders `_config.yml` `coda`) |
| `/colophon/` | `pages/colophon.md` | `default` | design/colophon |
| `/readme/` | `pages/readme.md` | `default` | socio-technical stack |
| `/ecofascism-bench/` | `pages/ecofascism-bench.md` | `default` | the eval-harness page |

**The landing → home split** is the core of the front-end design:

- **Landing** (`_layouts/landing.html`): a scroll-driven story in "acts" — ACT 1
  the bottle/exponential thought experiment (a `<canvas>` particle simulation),
  ACT 2 the Bartlett portrait + epigraph (particles morph into a stipple hedcut),
  ACT 3 the story bridge — ending in an **"Enter the book →"** link to `/home.html`.
  Driven by `assets/js/landing.js`.
- **Home hub** (`_layouts/home.html`): opens on the dark **title card**, then the
  two concepts (from `_config.yml` `registers`), then the chapter grid. A small
  inline script handles `.reveal` fade-ins, the title reveal, and the progress bar.

`landing.js` is written to **guard missing elements** (the title card / concepts /
chapters live only on the hub), so it no-ops cleanly on the landing page.

## Layouts & includes

- **Layouts** (`_layouts/`): `default.html` (base shell + `header`/`footer` for
  long-form pages), `landing.html`, `home.html`, `chapter.html`.
- **Includes** (`_includes/`):
  - `brandbar.html` — the immersive top bar on landing + hub. `mix-blend-mode:
    difference` keeps it legible over both the light canvas and the dark title
    card. Brand (upper-left) → `/home.html`; links from `site.nav`.
  - `header.html` — the frosted sticky nav for long-form/content pages. Same
    `site.nav` link set + a GitHub button. Brand → `/home.html`.
  - `footer.html` — content-page footer.
  - `chapter-card.html` / `chapter-span.html` — the chapter grid cards / full-width
    rows. `chapter-nav.html` — prev/next. `plate.html` — renders a chapter's SVG
    plate from `_data/plates.yml`. `head-extra.html` — favicons / head injection.

**Navigation is config-driven and single-sourced**: `_config.yml` → `nav`
(`Concepts` → `/home.html#concepts`, `Chapters` → `/home.html#chapters`,
`About` → `/about/`). Both `brandbar.html` and `header.html` loop over it. Change
links there, not in the includes.

## Content & data model

- **Chapters** — `_chapters/NN-slug.md`. Front matter: `slug`, `num`, `title`,
  optional `subtitle`/`abstract`, `source_note`; body is HTML/markdown. Publishes
  at `/chapters/<slug>/`.
- **Site identity & argument** — `_config.yml`:
  - `registers` — the **two operating concepts** (Quantitative Chauvinism,
    Lebensraum Imaginaries), rendered in the hub's `#concepts` section.
  - `coda` — `book` / `author` / `resources` blocks; rendered by the **About** page.
  - `nav`, `title`/`subtitle`/`description`, `theme_defaults`, `footer`.
- **`_data/*.yml`**:
  | File | Role | Hand-edit? |
  |------|------|-----------|
  | `cards.yml` | per-slug chapter-card copy (`era`/`concept`/`teaser`) | **No — generated** by `bin/apply-rename.py` |
  | `plates.yml` | per-slug SVG plate geometry (Bauhaus primitives in a 160×116 ratio viewBox) | generated, then hand-editable |
  | `spans.yml` | hand-authored appendix rows in the chapter grid (README / bench / colophon) | yes |
  | `parts.yml` | era/timeline "stepper" data | present but **not currently rendered** by any template after the nav rework |
  | `people` `orgs` `eras` `themes` `instruments` `documents` `prose` | "network reading" entity data | yes |
- **`_landing/*.md`** — a collection (`output: false`) of hand-authored prose
  snippets (concept definitions, epigraphs). Landing/definition prose is
  hand-authored, never machine-written (see HARNESS.md).

## Design system

The visual language is a deliberate **Bauhaus / constructivist** citation (the
reasoning is in `pages/colophon.md` — Weimar-era modernism vs. the machinery of
exclusion the book traces). Don't treat it as arbitrary styling.

- **Palette** (CSS vars in `assets/css/src/main.css`): `--ink` `#1a1a1a`,
  `--accent` brick-red `#b0382a`, `--accent-2` slate-blue `#285079`, `--yellow`
  `#e0a52f`, `--canvas` `#ece4d2`, plus `--clay`/`--mist`. Mirrored in
  `tailwind.config.js` (`ink`, `accent`, `blue`, `yellow`, `canvas`, …).
- **Type**: Josefin Sans (`--serif`, display), Archivo (`--sans`, running text),
  EB Garamond (`--emph`). **The "serif break" is semantic, not decorative**:
  italic emphasis and the book's charged terms switch to EB Garamond italic — the
  older, humanist voice surfacing through the modernist grid. Implemented via
  `.landing em,i,b,strong,…` and `.prose em,i` rules. Preserve this behavior.
- **Plates** are data-driven SVG (`_data/plates.yml` + `_includes/plate.html`),
  summonable anywhere by chapter slug.
- **Immersive scope**: both front-door pages use `<body class="landing">` to scope
  the bespoke scrollytelling CSS (the big unlayered block at the bottom of the
  source CSS). Content pages use the default Tailwind/typography styling.
- **Scroll reveals**: elements with class `.reveal` get `.is-on` via
  `IntersectionObserver` (JS-required; `prefers-reduced-motion` shows them
  immediately). CSS lives in the source file; new Tailwind utilities are picked up
  via the `content` globs in `tailwind.config.js`.

When editing CSS: most scrollytelling rules are **plain CSS** in the unlayered
section at the bottom of `assets/css/src/main.css` (not `@apply`), so they survive
purge regardless — but you still must `npm run build:css` to regenerate the dist.

## Draft-reconciliation harness (`bin/` + `metadata/` + `HARNESS.md`)

Keeps the website in sync with the evolving manuscript. **Read `HARNESS.md` before
touching `bin/`, `metadata/`, or any generated `_data` file.** Summary of the
pipeline (a new draft PDF lands in `source/`):

1. **`bin/ingest-draft.py`** — deterministic PDF → per-chapter extract
   (`metadata/v<N>/`, git-ignored). Has `--self-test`.
2. **`/ingest-draft`** (the `pdf-reader` agent) — populates
   `metadata/draft-metadata.json`, **the source of truth** (schema:
   `draft-metadata.schema.json`).
3. **`bin/audit-site.py`** — read-only audit of the live site vs. the interface →
   `metadata/audit-report.md` + `rename-plan.json`.
4. **`bin/apply-rename.py`** (`--apply`) — idempotent, semantic reconcile of
   `_config.yml` / `_data/*` / `_chapters/*` / redirects. Dry-run by default.

`bin/lint-site.py` is the offline pre-push gate. Slash commands `/ingest-draft`
and `/audit-constructs` wrap the stages. The rename is **semantic** (matches
construct phrases + machine ids, preserves look-alike terms like the *movement*
"ecofascism"), so never reconcile with `sed`.

## Conventions & gotchas

- **Don't commit unless asked.** If you do, branch off `master` for a PR.
- **Concurrent Cowork sessions** can change the working tree/branch mid-task
  (e.g. `bin/*.py` edits) — re-check `git status` before staging/committing and
  stage only your own files.
- **`assets/css/dist/` is gitignored** — your local rebuild is for validation;
  CI produces the deployed CSS. Always rebuild after editing the source CSS.
- **Generated data**: `_data/cards.yml` is written by the harness; reconcile via
  `metadata/draft-metadata.json`, not by hand.
- **Vanilla JS only** — no build step for JS; keep it framework-free and guarded
  against missing DOM nodes (pages share `landing.js`-style scripts).
- **No live local preview** of the rendered site (Jekyll won't build here); rely on
  the offline checks above + the CI build on your PR.
