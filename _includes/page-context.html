{%- comment -%} Emit window.PAGE_CONTEXT for the current page. The React app reads from this instead of from hard-coded literals so multiple scrollytelling pages (home, reading, chapter) can share one app shell. Resolves data via page frontmatter: page.page_kind → 'home' | 'reading' | 'chapter' | 'static' (default 'static') page.page_id → key for prose / scrolly lookups (default page_kind) page.scrolly_key → override scrolly file key (default page_id) page.prose_key → override prose file key (default page_id) page.highlight_part → part id pinned in the stepper (chapter pages) Steps merge the per-page nodes/focus with parts.yml metadata (title, num, start/end). A scrolly file may set `steps_inherit: ` to reuse another page's steps. Must run AFTER site-data.html (so window.GS_DATA is set) and BEFORE any .jsx. {%- endcomment -%} {%- assign page_kind = page.page_kind | default: "static" -%} {%- comment -%} page_id default: explicit `page.page_id`, else page.slug for chapter pages, else page_kind.
